Four sample programs are provided: two for lathes, one for 3-axis milling and another for 5-axis milling. Each of these samples includes the definition of a work piece, its tooling and a few sample manufacturing steps. Click on one of the images below to load the corresponding CATIA process (CATProcess) file.
 
This 2 axis turning sample performs roughing, spot drilling and deep drilling (both on center), followed by profile finishing, grooving and external threading operations. A 3 axis mill/turn sample does the same and also includes milling and drilling operations using the live spindle.
This 3 axis milling sample performs facing, pocketing, profiling and circular milling operations, followed by a series of spot drilling, deep drilling, boring, drilling, tapping and counter-sinking cycles.
This 5 axis milling sample performs multi-axis curve machining, multi-axis flank contouring and profile contouring (with vertical tool axis) operations, as well as a series of multi-axis spot drilling and drilling cycles arrayed around the outside perimeter of the flange.

Running a Sample

Each of these sample programs loads an ICAM PPTable, which defines the post-processor vocabulary that CATIA will output for standard operations. It is important to remember that an ICAM PPTable should always be loaded whenever using CAM-POST. You can do so by following these steps:
  1. Select the Part Operation object in the CATIA navigation list.
  2. Select the Edit menu, Part Operation submenu, Definition object.
  3. Select the icon of the NC machine on the window that appears.
  4. Choose one of the following from the Post-Processor words table list:
    ICAM_MILL_MM ICAM_LATHE_MM
    ICAM_MILL_INCH ICAM_LATHE_INCH
You can select either Generate NC Output in Batch Mode or Generate NC Output Interactively to call up the CAM-POST post-processor. When using the batch mode of operation, be sure to save your CATProcess first, since the batch mode processor uses the copy of the CATProcess last saved on disk, and not the one you are editing! You do not need to save your CATProcess when using the interactive mode of operation.

When the dialog window appears, select the In/Out tab to specify the manufacturing operations to post-process as well as the output format and file name.

A manufacturing program must be selected. Normally one is selected by default, unless manufacturing steps have not yet been defined.

The resulting NC data must be set to NC Code. The Output File must be specified with a complete path, which you can best do by selecting the Output File button instead of typing the file name directly into the field.
 
Select the NC Code tab to display a window that allows you to specify the post-processor name. A post-processor must be selected to allow the Execute button to be enabled.

A number of demonstration and/or production post-processors should be listed in the drop down menu. To obtain more information about a selected post-processor, press the [?] button. This will call up a description of the post-processor, including (for demonstration posts) the supported post-processor vocabulary and a sample NC program output.

CATIA Setup for Licensed CAM-POST Users

CATIA ships with a CAM-POST GENER executable, designed to run demonstration post-processors. Licensed CAM-POST users should run the kit/CATIA/setup.exe program, which is copied below the installation point when CAM-POST is installed. This setup program can be used to tell CATIA to run the version of CAM-POST received and installed from ICAM, instead of the default one that comes with the CATIA installation. WindowsTM users can find a link to the setup program in the Start/Programs/CAM-POST quick-start folder. A sample WindowsTM output follows:

Those users with Administrator privileges can set the default for all users of the computer. Select one of the "Use this version" check-boxes to have CATIA use the licensed CAM-POST version of GENER instead of the CATIA default.

Starting with CATIA V5 R12 SP4, it is possible to have a list of generated and dedicated post processors from a CAM-POST database appear in CATIA post-processor drop-down menus (in the Generate NC Code/NC Code dialog and also the Part Operation/Machine Editor dialog). This is done using the "Database" selections in the setup program. The default CAM-POST database or a selected CAM-POST database can be chosen.

If desired, the CATIA post-processor drop-downs can continue to show ICAM demonstration post processors. This is done by selecting the "Also include..." check-box. Note that demonstration post-processors with output limits will continue to enforce these limits, despite the presence of a CAM-POST license. To remove this limitation, the demonstration post-processor must be imported into a CAM-POST database. Demonstration post-processors can be found in the Startup/Manufacturing/ICAMPar/work directory below the CATIA installation point.
