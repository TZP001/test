SYNTAX

    ARCSLP / {ON|OFF}

      Controls the output of circular interpolation codes. ON allows circular interpolation codes to be output (default), OFF outputs circles using many small linear moves.

    AUXFUN / code ~[,code]

      Outputs one or more "L R F M" (miscellaneous function) codes. The post-processor makes no attempt to interpret the meaning or validity of the codes.

    CLAMP / [MANUAL,] [{ON|OFF},] a ,b ,c

      The a,b,c parameters specify the cldata x,y,z coordinates of the clamp position. ON moves the machine to the clamp position and then applies physical clamps. OFF removes clamps (if applicable) and then moves to the specified location. MANUAL inhibits output of any codes to the tape file.

    CLAMP / [MANUAL,] [{ON|OFF},] ~{ axis [,{ON|OFF}] [,value] }

      ON moves one or more machine axes to the specified machine coordinates and applies physical clamps. OFF removes clamps if applicable and then moves the machine. MANUAL inhibits output to the tape file. The initial ON or OFF specifies the default condition if ON or OFF is not explicitly stated for each axis. If value is omitted then the current location is used. Axis can be one of the following:

        Axis keyword Machine axis Physical clamps
        XAXISXNot available
        YAXISYNot available
        ZAXISZNot available

    CLEARP / a ,b ,c ,d

      Specifies a clearance plane to be used with RETRCT in terms of the (a,b,c,d) coefficients of a plane equation. Note that the plane is defined in the CL reference system and not the machine system.

    CLEARP / [{XYPLAN|YZPLAN|ZXPLAN},] d

      Specifies a clearance plane to be used with RETRCT, where d is the distance of the clearance plane above the named CL plane (XYPLAN by default). Note that d is measured in CL units and not machine units.

    CLRSRF / x ,y ,z

      Specifies a clearance point to be used with RETRCT. Note that the point is defined in the CL reference system and not the machine system. The tool tip will be moved onto a plane passing through the point and perpendicular to the tool axis.

    CLRSRF / a ,b ,c ,d

      Specifies a clearance plane to be used with RETRCT in terms of the (a,b,c,d) coefficients of a plane equation. Note that the plane is defined in the CL reference system and not the machine system.

    CLRSRF / x ,y ,z ,i ,j ,k ,r

      Specifies a clearance cylinder to be used with RETRCT in terms of a centre point (x,y,z), axis orientation (i,j,k) and radius (r). Note that the cylinder is defined in the CL reference system and not the machine system.

    COOLNT / {ON|OFF}

      Starts or stops the flow of coolant. ON starts the last specified coolant type (FLOOD by default), OFF stops the coolant flow with a L R F M9 code.

    COOLNT / FLOOD

      Outputs a L R F M8 code to start the flow of flood coolant.

    CUTCOM / {ON|OFF}

      Controls diameter compensation. ON reinstates the last diameter compensation specified, OFF cancels diameter compensation by outputting a L code.

    CUTCOM / {LEFT|RIGHT}

      Activates diameter compensation by outputting either a L or a L code.

    CYCLE / {ON|OFF|NOMORE}

      Activates or cancels the last specified cycle type.

    CYCLE / type, dimension, feed [,clearance] [,options]

      This is the general cycle format. The specified cycle will be performed at each non-RAPID GOTO that follows. The cycle is cancelled by a CYCLE/OFF or CYCLE/NOMORE.

      The type parameter is a minor word identifying one of a series of standard cycles. Dimension information defines the total depth of the hole. The feed defines the cutting rate of the cycle, and must be specified as follows:

        {IPM|IPR|MMPM|MMPR|PERMIN|PERREV} ,value

      The optional clearance value specifies the distance above the point that the feed motion of the cycle should start. The clearance is modal from one cycle to the next. The default clearance is 3.

      The options depend on the cycle type. The following cycle types and options are supported:

        TypeOptions
        BORERAPTO,a DWELL[,REV],b ORIENT,c
        BRKCHPRAPTO,a TIMES,b DECR,c
        CSINKRAPTO,a DWELL[,REV],b
        DEEPRAPTO,a DECR,b
        DRILLRAPTO,a DWELL[,REV],b
        FACERAPTO,a DWELL[,REV],b
        MANUALNone
        REAMRAPTO,a DWELL[,REV],b
        TAPRAPTO,a INVERS
        THRURAPTO,a

      The [,RAPTO,a] option defines an additional distance a to plunge at rapid starting from the clearance plane.

      The [,DWELL[,REV],a] option defines a dwell time to be applied once the full cycle depth is reached. The dwell can be specified in either seconds or revolutions (using REV). Dwell times are output using the P register. Valid times range from .001 through 99.999 in increments of .001. The default time, if omitted, is 1 seconds.

      The [,INVERS] option specifies a left hand (CCLW) tapping cycle.

      The [,ORIENT[,a]] option is used to generate a fine boring cycle, where the cutting tip is oriented to a degrees (0 degrees default), moved off the side of the hole, and then retracted. A BORE with ORIENT cycle is not supported directly on your machine. The post processor will simulate the feature.

      The [,TIMES,a] option causes the tool to perform a full retract every a pecks during a BRKCHP cycle.

    CYCLE / BORE ,depth ,feed [,clear]

      This is a basic BORE cycle, which performs a feed to depth, spindle stop, rapid retract and spindle restart. The cycle is emulated by the post-processor.

    CYCLE / BORE ,depth ,feed [,clear] ,DWELL[,REV],a

      This is a basic BORE with dwell cycle, which performs a feed to depth, dwell, spindle stop, rapid retract and spindle restart. The cycle is emulated by the post-processor.

    CYCLE / BORE ,depth ,feed [ ,clear [,jog] ] ,ORIENT[,a]

      This is a basic BORE with orient cycle, which performs a feed to depth, spindle orient, jog off center, rapid retract, jog to center and spindle restart. The default jog distance is .3 . The cycle is emulated by the post-processor.

    CYCLE / BRKCHP [,depth,INCR] ~{ ~,pecks,feed } [,clear]

      This cycle drills a hole with intermittent small retracts to inhibit formation of long chips. The pecks parameter is a series of delta Z amounts (if INCR is coded) or absolute depths. The cycle is emulated by the post-processor.

    CYCLE / CSINK ,odia ,ang [,idia] ,feed [,clear]

      This is a countersinking cycle. The odia parameter specifies the outside diameter of the countersink. The ang parameter is the full included angle of the countersinking tool. The idia is an optional parameter specifying the inside diameter of the hole (GENER uses this information to accelerate the countersinking cycle). A L code is output for this cycle.

    CYCLE / CSINK ,odia ,ang [,idia] ,feed [,clear] ,DWELL[,REV],a

      This is a countersinking cycle with dwell. The odia parameter specifies the outside diameter of the countersink. The ang parameter is the full included angle of the countersinking tool. The idia is an optional parameter specifying the inside diameter of the hole (GENER uses this information to accelerate the countersinking cycle). A L code is output for this cycle.

    CYCLE / DEEP [,depth,INCR] ~{ ~,pecks,feed } [,clear]

      This cycle drills a hole in steps, with full retracts to the clearance plane to clear chips from the hole. The pecks parameter is a series of delta Z amounts (if INCR is coded) or absolute depths. A L code is output for this cycle.

    CYCLE / DRILL ,depth ,feed [,clear]

      This is a basic DRILL cycle, which performs a feed to depth and rapid retract. A L code is output for this cycle.

    CYCLE / FACE ,depth ,feed [,clear] ,DWELL[,REV],a

      This is a basic DRILL cycle with dwell, which performs a feed to depth, dwells for a period of a seconds or a revolutions, and then does a rapid retract. (Note that DRILL can be specified instead of FACE, with the same results.) A L code is output for this cycle.

    CYCLE / MANUAL [,clear]

      The tool will be brought to the specified clearance at each point in the cycle and a STOP command output.

    CYCLE / REAM ,depth ,feed [,clear]

      This is a basic REAM cycle, which performs a feed to depth and a feed retract. The cycle is emulated by the post-processor.

    CYCLE / REAM ,depth ,feed [,clear] ,DWELL[,REV],a

      This is a basic REAM cycle with dwell, which performs a feed to depth, dwells for a period of a seconds or a revolutions, and then does a feed retract. The cycle is emulated by the post-processor.

    CYCLE / TAP ,depth ,feed [,clear]

      This is a basic RH thread TAP cycle, which performs a spindle CLW, feed to depth, spindle CCLW, feed retract, and a spindle CLW. An additional TPI feed type is permitted with the TAP cycle to define the feed in threads per inch. A L code is output for this cycle.

    CYCLE / TAP ,depth ,feed [,clear] ,INVERS

      This is a basic LH thread TAP cycle, which performs a spindle CCLW, feed to depth, spindle CLW, feed retract, and a spindle CCLW. An additional TPI feed type is permitted with the TAP cycle to define the feed in threads per inch. A L code is output for this cycle.

    CYCLE / THRU ~{,depths} ,feed [,clear]

      This is a web drilling cycle. An odd number of depths must be specified. The first depth specifies the depth of the first web. The second depth is the clearance above the second web. The third is the depth of the second web. Etcetera. The tool is only retracted when the final depth is reached. The cycle is emulated by the post-processor.

    DELAY / seconds

      Outputs a ""CYCL DEF 9.1 DWELL "" register to halt the machine for a specified time. Any delay between .001 and 99.999 in increments of .001 can be specified.

    DELAY / REV,revolutions

      Outputs a delay in seconds that approximates the number of spindle revolutions specified (see "DELAY/seconds" above).

    DISPLY / string ~[,value]

      Formats the optional values into the character string and outputs the string to the NC tape file in the following format:

        ===string====

      The machine CRT can display messages up to 80 characters long. Longer messages will be broken into 2 or more messages. A maximum of 1 messages can be seen on the display at any one time.

    DISPLY / {ON|OFF}

      Specifies whether or not subsequent PPRINT text should be output as display messages on the CRT.

    END

      The END command outputs a L R F M2 to signal the machine that it has reached the end of the program. The spindle and coolant are automatically turned off by the L R F M2 code.

    FEDRAT / [value] [,{IPM|IPR|MMPM|MMPR|PERMIN|PERREV}]

      Specifies an interpolation rate to be used for subsequent motions. The feed types IPM, MMPM and PERMIN all specify a rate in terms of units per minute, where units is one of inches, millimeters or current cl units. The feed types IPR, MMPR and PERMIN specify a rate in terms of units per spindle revolution. The feed type is modal, and defaults to PERMIN if not specified.

    INSERT / string ~[,value]

      Formats the optional values into the character string and outputs the string to the NC tape file as a command block. A sequence number will be prefixed to the string. The post-processor makes no attempt to interpret the meaning or validity of the inserted tape block.

    LIMIT / [ON|OFF] [,zone] ~{ ,axis ,limit }

      Controls travel limit checking. ON (default) and OFF indicate whether travel limit checking should be activated or deactivated for the specified travel zone (1 by default). Up to 10 zones can be defined. Axis specifies a machine axis by keyword as follows:

        Axis keywordMachine axis
        XAXISX
        YAXISY
        ZAXISZ

      Limit specifies the type of travel limit checking desired, and can be one of the following:

        [,ON],low,high
        [,ON],total
        ,OFF

      Where low and high specify the minimum and maximum travel limits, total specifies the travel limits in terms of total displacement, and OFF deactivates travel checking for the named axis. By default, no travel limit checking is performed by the post-processor.

    LOADTL / tool [,options]

      Tool specifies the tool to load, which must be between 1 and 100. The tool number is output using the ""TOOL CALL " " Z"" register. The options are identical to those available with the TOOLNO command, and are described there.

    MACHIN / HEIDN5, 30 ~[,{keyword|value|string}]

      This command identifies the post-processor. It is not required. Any additional parameters are ignored by GENER.

    MODE / INCR, {ON|OFF}

      Changes the positioning mode between absolute and incremental. ON selects incremental mode, OFF selects absolute mode. The machine interprets the current mode by the register name being used for positioning (eg. X for absolute, "IX" for incremental). This command affects the output tape codes, it does not affect the post-processors interpretation of cl data information.

    OPSTOP

      There is no optional stop capability on the machine. Instead, the OPSTOP command is processed as a STOP command, which outputs a L R F M0 to halt the machine. The spindle is automatically turned off by the L R F M0 code. The coolant is turned off by the post-processor (if on) before outputting the L R F M0 code. The spindle and coolant are restarted by the post-processor (if they were previously on) in the tape blocks immediately following the L R F M0 code.

    ORIGIN / a ,b ,c

      The a,b,c parameters specify the location of the machine origin in terms of cl file x,y,z coordinates.

    ORIGIN / matrix

      Specifies the machine origin in terms of a matrix defined in the cl file coordinate system. The matrix is defined by 12 numeric values which represent the location and orientation of the machine reference system in terms of the cl file reference system.

    PARTNO / string ~[,value]

      Formats the optional values into the character string and uses the string as a program identification. The program identification is output at the top of each page of the post-processor verification file. It is also be output as "man-readable" characters at the start and end of the tape file. If a PARTNO command is not found within the first 20 records of the cl file, the default identification "Undefined PARTNO" will be used.

    PPRINT / string ~[,value]

      Formats the optional values into the character string and outputs the string either to the post-processor listing file (default) or to the operator console. The target is selected using the DISPLY command. Coding DISPLY/ON selects the operator console for subsequent PPRINT messages; coding DISPLY/OFF (the default) selects the post-processor listing file for PPRINT messages.

    PREFUN / code ~[,code]

      Outputs one or more L (preparatory function) codes. The post-processor makes no attempt to interpret the meaning or validity of the codes.

    RAPID

      Causes the next motion to be performed at the maximum positioning rate of the machine. Feedrate mode is immediately reinstated following the rapid motion (a FEDRAT command can be programmed to cancel the pending rapid move).

    RETRCT

      Retracts the tool, along the tool axis, to the clearance plane defined by the last CLEARP or CLRSRF command. No movement will be made if the tool is already at or above the clearance plane. The retract motion will be made at rapid.

    REWIND / value

      The REWIND command outputs a L R F M30 to signal the machine that it has reached the end of the program and that the program should be reset to the start. The value is ignored, but required for historic reasons. The spindle and coolant are automatically turned off by the L R F M30 code.

    SEQNO / AUTO

      Causes subsequent tape block sequence numbers to match the cl data record number that originated the NC instruction. If a cl record produces multiple tape blocks, then each block will have the same number.

    SEQNO / [CONST,] value

      If CONST is omitted, the next tape block will be output with sequence number value. Sequence numbering then reverts back to the previously specified format. If CONST is specified, then all subsequent blocks will be numbered value until a new SEQNO command is coded.

    SEQNO / [start,] INCR,incr

      Incr specifies the amount to increase the sequence number by each time it is output. Start specifies a new starting sequence number. If omitted, it defaults to the last seqence value plus the new increment. Sequence numbers are output using the """" register, and can range from 0 through 999999 in increments of 1 . By default, sequence numbers will start at 0 and increment by 1 for each block output.

    SPINDL / {ON|OFF}

      Controls the spindle. ON starts the spindle at the last specified rate. OFF stops the spindle.

    SPINDL / ORIENT [,angle]

      Outputs a L R F M17 code to stop and orient the tool to the specified angle in degrees (if omitted, then the last specified angle will be used, or if none then 0 degrees). The ""CYCL DEF 13.1 ANGLE "" is used to control the orientation. Any angle from 0 through 359.999 in steps of .001 degrees can be specified.

    SPINDL / [s] [,{RPM|SFM|SMM}] [,{CLW|CCLW}]

      Specifies a spindle speed s in revolutions per minute (RPM), surface feet per minute (SFM) or surface meters per minute (SMM). If the type is omitted, then the last specified type will be used (RPM by default). If the speed is omitted, then the last specified speed will be used (minimum speed by default). The speed is output using the S register. The spindle speed can range from 1 through 9999.

      The CLW and CCLW parameters specify the spindle rotation direction. A L R F M3 is output for clockwise rotation. A L R F M4 is output for counter-clockwise rotation.

    STOP

      The STOP command outputs a L R F M0 to halt the machine. The spindle is automatically turned off by the L R F M0 code. The coolant is turned off by the post-processor (if on) before outputting the L R F M0 code. The spindle and coolant are restarted by the post-processor (if they were previously on) in the tape blocks immediately following the L R F M0 code.

    TRANS / {ON|OFF}

      ON reinstates the last specified axes shifts, OFF removes the accumulated axes shifts. Axes shifts are entered using the other forms of the TRANS command listed below. Note that axes shifts are not accumulative. For example, coding TRANS/XAXIS,3 and then TRANS/XAXIS,5 will result in a shift of 5 along the x axis. Each new TRANS shift replaces the old one.

    TRANS / [ON,] a ,b ,c

      The a,b,c parameters specify the cldata x,y,z delta translation amounts. Note that the translation is specified as a delta amount in cl file coordinates. ON need not be specified since it is the default.

    TRANS / [{ON|OFF},] ~{ axis [,{ON|OFF}] [,value] }

      The initial ON or OFF specifies the the default condition if ON or OFF is not explicitly stated for each axis. ON shifts the named machine axis by value machine units (not cl file units). OFF removes any translation (do not specify a value with the OFF format). Axis specifies a machine axis by keyword as follows:

        Axis keywordMachine axis
        XAXISX
        YAXISY
        ZAXISZ
