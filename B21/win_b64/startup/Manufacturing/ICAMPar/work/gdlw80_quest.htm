SYNTAX

    ARCSLP / {ON|OFF}

      Controls the output of circular interpolation codes. ON allows circular interpolation codes to be output (default), OFF outputs circles using many small linear moves.

    AUXFUN / code ~[,code]

      Outputs one or more M (miscellaneous function) codes. The post-processor makes no attempt to interpret the meaning or validity of the codes.

    CLAMP / [MANUAL,] [{ON|OFF},] a ,b

      The a,b parameters specify the cldata x,y coordinates of the clamp position. ON moves the machine to the clamp position and then applies physical clamps. OFF removes clamps (if applicable) and then moves to the specified location. MANUAL inhibits output of any codes to the tape file.

    CLAMP / [MANUAL,] [{ON|OFF},] ~{ axis [,{ON|OFF}] [,value] }

      ON moves one or more machine axes to the specified machine coordinates and applies physical clamps. OFF removes clamps if applicable and then moves the machine. MANUAL inhibits output to the tape file. The initial ON or OFF specifies the default condition if ON or OFF is not explicitly stated for each axis. If value is omitted then the current location is used. Axis can be one of the following:

        Axis keyword Machine axis Physical clamps
        XAXISXNot available
        ZAXISZNot available

    CLEARP / d

      Specifies a clearance plane to be used with RETRCT, where d is the distance along the machine Z axis from the part coordinate origin. Note that d is measured in CL units and not machine units.

    COOLNT / {ON|OFF}

      Starts or stops the flow of coolant. ON starts the last specified coolant type (FLOOD by default), OFF stops the coolant flow with a M09 code.

    COOLNT / FLOOD

      Outputs a M08 code to start the flow of flood coolant.

    CUTCOM / {ON|OFF}

      Controls diameter compensation. ON reinstates the last diameter compensation specified, OFF cancels diameter compensation by outputting a G40 code.

    CUTCOM / OFF ,LENGTH [,{NOW|NEXT}]

      Cancels length compensation by outputting a D000 code.

    CUTCOM / ON ,LENGTH [,offset] [,{NOW|NEXT}]

      Activates length compensation by outputting a D register containing the specified (or last) offset value. There are 999 offsets available.

    CUTCOM / {LEFT|RIGHT}

      Activates diameter compensation by outputting either a G41 or a G42 code.

    CYCLE / {ON|OFF|NOMORE}

      Activates or cancels the last specified cycle type.

    CYCLE / type, dimension, feed [,clearance] [,options]

      This is the general cycle format. The specified cycle will be performed at each non-RAPID GOTO that follows. The cycle is cancelled by a CYCLE/OFF or CYCLE/NOMORE.

      The type parameter is a minor word identifying one of a series of standard cycles. Dimension information defines the total depth of the hole. The feed defines the cutting rate of the cycle, and must be specified as follows:

        {IPM|IPR|MMPM|MMPR|PERMIN|PERREV} ,value

      The optional clearance value specifies the distance above the point that the feed motion of the cycle should start. The clearance is modal from one cycle to the next. The default clearance is .1.

      The options depend on the cycle type. The following cycle types and options are supported:

        TypeOptions
        BORERAPTO,a DWELL[,REV],b
        BRKCHPRAPTO,a TIMES,b
        CSINKRAPTO,a DWELL[,REV],b
        DEEPRAPTO,a
        DRILLRAPTO,a DWELL[,REV],b
        FACERAPTO,a DWELL[,REV],b
        MANUALNone
        REAMRAPTO,a DWELL[,REV],b

      The [,RAPTO,a] option defines an additional distance a to plunge at rapid starting from the clearance plane.

      The [,DWELL[,REV],a] option defines a dwell time to be applied once the full cycle depth is reached. The dwell can be specified in either seconds or revolutions (using REV). Dwell times are set manually by the operator, so the time specified is only used for post-processor time calculations. The default time, if omitted, is 2 seconds.

      The [,TIMES,a] option causes the tool to perform a full retract every a pecks during a BRKCHP cycle.

    CYCLE / BORE ,depth ,feed [,clear]

      This is a basic BORE cycle, which performs a feed to depth, spindle stop, rapid retract and spindle restart. The cycle is emulated by the post-processor.

    CYCLE / BORE ,depth ,feed [,clear] ,DWELL[,REV],a

      This is a basic BORE with dwell cycle, which performs a feed to depth, dwell, spindle stop, rapid retract and spindle restart. The cycle is emulated by the post-processor.

    CYCLE / BRKCHP [,depth,INCR] ~{ ~,pecks,feed } [,clear]

      This cycle drills a hole with intermittent small retracts to inhibit formation of long chips. The pecks parameter is a series of delta Z amounts (if INCR is coded) or absolute depths. The cycle is emulated by the post-processor.

    CYCLE / CSINK ,odia ,ang [,idia] ,feed [,clear]

      This is a countersinking cycle. The odia parameter specifies the outside diameter of the countersink. The ang parameter is the full included angle of the countersinking tool. The idia is an optional parameter specifying the inside diameter of the hole (GENER uses this information to accelerate the countersinking cycle). The cycle is emulated by the post-processor.

    CYCLE / CSINK ,odia ,ang [,idia] ,feed [,clear] ,DWELL[,REV],a

      This is a countersinking cycle with dwell. The odia parameter specifies the outside diameter of the countersink. The ang parameter is the full included angle of the countersinking tool. The idia is an optional parameter specifying the inside diameter of the hole (GENER uses this information to accelerate the countersinking cycle). The cycle is emulated by the post-processor.

    CYCLE / DEEP [,depth,INCR] ~{ ~,pecks,feed } [,clear]

      This cycle drills a hole in steps, with full retracts to the clearance plane to clear chips from the hole. The pecks parameter is a series of delta Z amounts (if INCR is coded) or absolute depths. The cycle is emulated by the post-processor.

    CYCLE / DRILL ,depth ,feed [,clear]

      This is a basic DRILL cycle, which performs a feed to depth and rapid retract. The cycle is emulated by the post-processor.

    CYCLE / FACE ,depth ,feed [,clear] ,DWELL[,REV],a

      This is a basic DRILL cycle with dwell, which performs a feed to depth, dwells for a period of a seconds or a revolutions, and then does a rapid retract. (Note that DRILL can be specified instead of FACE, with the same results.) The cycle is emulated by the post-processor.

    CYCLE / MANUAL [,clear]

      The tool will be brought to the specified clearance at each point in the cycle and a STOP command output.

    CYCLE / REAM ,depth ,feed [,clear]

      This is a basic REAM cycle, which performs a feed to depth and a feed retract. The cycle is emulated by the post-processor.

    CYCLE / REAM ,depth ,feed [,clear] ,DWELL[,REV],a

      This is a basic REAM cycle with dwell, which performs a feed to depth, dwells for a period of a seconds or a revolutions, and then does a feed retract. The cycle is emulated by the post-processor.

    DELAY / seconds

      Outputs a G04 code to cause a delay in seconds. The X register specifies a delay time between .0001 and 9999.9999 in increments of .0001.

    DELAY / REV,revolutions

      Outputs a delay in seconds that approximates the number of spindle revolutions specified (see "DELAY/seconds" above).

    DISPLY / string ~[,value]

      Formats the optional values into the character string and outputs the string to the NC tape file in the following format:

        N9999 (MSG,string)

      The machine CRT can display messages up to 80 characters long. Longer messages will be broken into 2 or more messages. A maximum of 1 messages can be seen on the display at any one time.

    DISPLY / {ON|OFF}

      Specifies whether or not subsequent PPRINT text should be output as display messages on the CRT.

    END

      The END command outputs a M02 to signal the machine that it has reached the end of the program. The spindle and coolant are automatically turned off by the M02 code.

    FEDRAT / [value] [,{IPM|IPR|MMPM|MMPR|PERMIN|PERREV}]

      Specifies an interpolation rate to be used for subsequent motions. The feed types IPM, MMPM and PERMIN all specify a rate in terms of units per minute, where units is one of inches, millimeters or current cl units. The feed types IPR, MMPR and PERMIN specify a rate in terms of units per spindle revolution. The feed type is modal, and defaults to PERMIN if not specified.

    INSERT / string ~[,value]

      Formats the optional values into the character string and outputs the string to the NC tape file as a command block. A sequence number will be prefixed to the string. The post-processor makes no attempt to interpret the meaning or validity of the inserted tape block.

    LIMIT / [ON|OFF] [,zone] ~{ ,axis ,limit }

      Controls travel limit checking. ON (default) and OFF indicate whether travel limit checking should be activated or deactivated for the specified travel zone (1 by default). Up to 10 zones can be defined. Axis specifies a machine axis by keyword as follows:

        Axis keywordMachine axis
        XAXISX
        ZAXISZ

      Limit specifies the type of travel limit checking desired, and can be one of the following:

        [,ON],low,high
        [,ON],total
        ,OFF

      Where low and high specify the minimum and maximum travel limits, total specifies the travel limits in terms of total displacement, and OFF deactivates travel checking for the named axis. By default, no travel limit checking is performed by the post-processor.

    MACHIN / GDLW, 80 ~[,{keyword|value|string}]

      This command identifies the post-processor. It is not required. Any additional parameters are ignored by GENER.

    MODE / INCR, {ON|OFF}

      Changes the positioning mode between absolute and incremental. ON outputs a G91 code to select incremental mode, OFF outputs a G90 code to select absolute mode. This command affects the output tape codes, it does not affect the post-processors interpretation of cl data information.

    OFSTNO / {value|NOMORE}

      Specifies a new tool correction offset value, or with NOMORE, cancels tool correction. This command has the same affect as coding "TURRET/tool,value" where "tool" is the currently loaded tool number.

    OPSKIP / {ON|OFF}

      Controls the output of the block delete code "/". ON causes the block delete to be output with subsequent tape blocks, OFF resumes normal tape output.

    OPSTOP

      The OPSTOP command outputs a M01 to optionally halt the machine depending on the state of a switch setting at the machine.

    ORIGIN / a ,b

      The a,b parameters specify the location of the machine origin in terms of cl file x,y coordinates.

    ORIGIN / matrix

      Specifies the machine origin in terms of a matrix defined in the cl file coordinate system. The matrix is defined by 12 numeric values which represent the location and orientation of the machine reference system in terms of the cl file reference system.

    PARTNO / string ~[,value]

      Formats the optional values into the character string and uses the string as a program identification. The program identification is output at the top of each page of the post-processor verification file. It is also be output as "man-readable" characters at the start and end of the tape file. If a PARTNO command is not found within the first 20 records of the cl file, the default identification "Undefined PARTNO" will be used.

    POSTN / [MANUAL,] {ON|OFF}

      ON presets the machine coordinate displays to the last specified preset position by outputting a G92 code with the appropriate axes values. This has the effect of shifting the machine coordinates. OFF removes the effect of the most recent POSTN command by outputting the reverse shift. If MANUAL is specified, then no tape codes are output, however the post-processor assumes the shift has been done at the machine.

    POSTN / [MANUAL,] [ON,] a ,b

      The a,b parameters define the new machine display coordinate location in terms of the specified cldata x,y coordinates. Coordinate shifting is accumulative. Note that ON need not be specified since ON is the default. If MANUAL is specified, then no tape codes are output, however the post-processor assumes the shift has been done at the machine.

    POSTN / [MANUAL,] [{ON|OFF},] ~{ axis [,{ON|OFF}] [,value] }

      ON presets an axis to a specified value in machine coordinates (ie. no conversion from cl to machine is done). OFF removes the preset amount from the specified axis ( value must not be specified with the OFF format). The initial ON or OFF specifies the default condition if ON or OFF is not explicitly stated for each axis. Axis can be one of the following:

        Axis keywordMachine axis
        XAXISX
        ZAXISZ

      If MANUAL is specified, then no tape codes are output, however the post-processor assumes the shift has been done at the machine.

    PPRINT / string ~[,value]

      Formats the optional values into the character string and outputs the string either to the post-processor listing file (default) or to the operator console. The target is selected using the DISPLY command. Coding DISPLY/ON selects the operator console for subsequent PPRINT messages; coding DISPLY/OFF (the default) selects the post-processor listing file for PPRINT messages.

    PREFUN / code ~[,code]

      Outputs one or more G (preparatory function) codes. The post-processor makes no attempt to interpret the meaning or validity of the codes.

    RAPID

      Causes the next motion to be performed at the maximum positioning rate of the machine. Feedrate mode is immediately reinstated following the rapid motion (a FEDRAT command can be programmed to cancel the pending rapid move).

    RETRCT

      Retracts the tool, parallel to the spindle centerline, to the clearance plane specified by the last CLEARP command. No movement will be made if the tool is already at or above the clearance plane. The retract motion will be made at rapid.

    REWIND / value

      The REWIND command outputs a M30 to signal the machine that it has reached the end of the program and that the program should be reset to the start. The value is ignored, but required for historic reasons. The spindle and coolant are automatically turned off by the M30 code.

    SELCTL / tool [,options]

      Tool specifies the tool to pre-select, which must be between 1 and -246811958. The tool number is selected using the T register. Tool pre-selection is optional. If omitted, GENER will automatically select the next tool immediately following a tool change. The options are identical to those available with the TOOLNO command, and are described there.

    SEQNO / AUTO

      Causes subsequent tape block sequence numbers to match the cl data record number that originated the NC instruction. If a cl record produces multiple tape blocks, then each block will have the same number.

    SEQNO / [CONST,] value

      If CONST is omitted, the next tape block will be output with sequence number value. Sequence numbering then reverts back to the previously specified format. If CONST is specified, then all subsequent blocks will be numbered value until a new SEQNO command is coded.

    SEQNO / [start,] INCR,incr

      Incr specifies the amount to increase the sequence number by each time it is output. Start specifies a new starting sequence number. If omitted, it defaults to the last seqence value plus the new increment. Sequence numbers are output using the N register, and can range from 1 through 9999 in increments of 1 . By default, sequence numbers will start at 1 and increment by 1 for each block output.

    SPINDL / {ON|OFF}

      Controls the spindle. ON starts the spindle at the last specified rate. OFF stops the spindle.

    SPINDL / NEUTRL

      Outputs a M38 code to stop and disengage the spindle from the spindle motor (ie. puts the spindle into neutral).

    SPINDL / [s] [,{RPM|SFM|SMM}] [,{CLW|CCLW}] [,MAXRPM{,{max|OFF}]

      Specifies a spindle speed s in revolutions per minute (RPM), surface feet per minute (SFM) or surface meters per minute (SMM). If the type is omitted, then the last specified type will be used (RPM by default). If the speed is omitted, then the last specified speed will be used (minimum speed by default). The speed is output using the S register. The spindle speed can range from 1 through 9999.

      The CLW and CCLW parameters specify the spindle rotation direction. A M03 is output for clockwise rotation. A M04 is output for counter-clockwise rotation.

    STOP

      The STOP command outputs a M00 to halt the machine.

    TRANS / {ON|OFF}

      ON reinstates the last specified axes shifts, OFF removes the accumulated axes shifts. Axes shifts are entered using the other forms of the TRANS command listed below. Note that axes shifts are not accumulative. For example, coding TRANS/XAXIS,3 and then TRANS/XAXIS,5 will result in a shift of 5 along the x axis. Each new TRANS shift replaces the old one.

    TRANS / [ON,] a ,b

      The a,b parameters specify the cldata x,y delta translation amounts. Note that the translation is specified as a delta amount in cl file coordinates. ON need not be specified since it is the default.

    TRANS / [{ON|OFF},] ~{ axis [,{ON|OFF}] [,value] }

      The initial ON or OFF specifies the the default condition if ON or OFF is not explicitly stated for each axis. ON shifts the named machine axis by value machine units (not cl file units). OFF removes any translation (do not specify a value with the OFF format). Axis specifies a machine axis by keyword as follows:

        Axis keywordMachine axis
        XAXISX
        ZAXISZ

    TURRET / tool [ offset [,x,y] ] [,options]

      Tool specifies the tool to load, which must be between 1 and 999. The tool number is output using the T register. A M06 code is also output. The options are identical to those available with the TOOLNO command, and are described there. When TURRET is coded, the post-processor will automatically stop the coolant flow and stop the spindle.
