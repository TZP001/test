Introduction

Thank you for choosing CAM-POST, the NC post processing Product from ICAM Technologies Corporation. CAM-POST is a post-processor generator embedded within the CATIA V5 architecture to generate NC Data output. It can accommodate a wide variety of machine types having multiple axis path control. This document describes the steps to install, upgrade and configure CAM-POST to run in an integrated CATIA V5 environment. This document is divided into the following sections:

CATIA V5 NC manufacturing Default Settings
This section describes the steps to set up the CATIA V5 NC Manufacturing environment to integrate with the ICAM post processors.

ICAM Technical Support
ICAM Technical Support and contact details are listed in this section

 

1          CATIA V5 NC Manufacturing Default Settings

Under the CATIA menu Tools > Options > Machining > Output,

Set the post-processor option to ICAM

 

 

1.1        CATIA V5 ICAM PPTables

The default CATIA PPTables directory should have the following 2 ICAM PPTable files.

·         ICAM_INCH.pptable             (imperial units)

·         ICAM_MM.pptable                (metric units)

The default installation directory for the CATIA V5 PPTables is typically:

C:\Program Files\Dassault Systemes\BXX\intel_a\startup\Manufacturing\PPTables

1.2        CATIA V5 / CAM-POST - Environment Setup

Run the CAM-POST/CATIA integration setup utility to set CAM-POST to run within CATIA V5. The setup can be run from the start menu:

Start > Programs > ICAM Productivity Tools > Setup > Kits > CATIA V17

Or you can run the setup utility (contained inside CAM-POST root directory) via Windows Explorer. Typically:

C:\Program Files\CAMPOST V17\Kit\CATIA\Win32\Setup.exe

Select the options for running Gener (you can set it for the current user only or for all user of this computer) and browse for the database containing the post-processors. By default, you should use: C:\Program Files\CAMPOST V17\work\campost.dbf database. If you want to get back to the initial Dassault Systeme installation of CAM-POST, you need to re-run the setup.exe file and uncheck the boxes from the setup panel. You can also select a special interface kit, if one was previously created. By default, this field should point to Automatic or CATIA.

 

1.3        CATIA Part Operation Setup:

In the CATIA PPR Tree, double click the Part Operation icon. In the Part Operation dialog, select the Machine icon

In the Machine Editor dialog, select the machine type, your custom post-processor and corresponding ICAM post-processor words table. You can also immediately set the NC data type to ISO.

 

Next, select NC data format as Point (X,Y,Z) for 3 Axis output or Axis (X,Y,Z,I,J,K) for Multi-Axis output. Set all other options as shown in the window on the right. If you have NURBS licensed post-processor, check 3D Nurbs interpolation option if you want to use NURBS output.

1.5        Post Processing Options and Information

The NC data output from CATIA through the ICAM post-processor can be executed either in silent mode or in a graphical display mode.

 

By default the NC Code generation is executed in a silent mode. To post process in a graphical display mode, simply check the More Options button.

To view information on the post-processor being executed, simply click on the ? Button. This will open up a help page with more information and links on this and other post-processors.
