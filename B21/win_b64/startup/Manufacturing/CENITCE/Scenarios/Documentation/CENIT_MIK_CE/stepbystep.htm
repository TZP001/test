Step by Step

This section shows, how to configure MPS and how to run the ISO Code based simulation. The scenario shows how to work with different tool offsets and how to detect a clash.

There are 2 processes in this scenario. The process ZIMMERMANN_4711061160_NC01_START.CATProcess is the process for training.  The ZIMMERMANN_4711061160_NC01_END.CATProcess is for demonstration (with predefined values/ clash detection and loaded machine). There are 2 Manufacturing programs in each process:  “Manufacturing Program.1 (to be controlled)” has some clash problems to solve and “Manufacturing Program.2 (ok)” is the corrected program.

Configure Scenario

This section describes the necessary configuration settings for ISO Code based simulation

•Common Settings

Left-click on Menu Item "Tools" then "Options...". Select "Machining" then "Output" Tabpage.

Set Option "PostProcessor" to "Cenit".

 

 

•MPS Definitions on CATProcess

Load the CATProcess with left-click on Menu "File" then "Open..."

Select the file ZIMMERMANN_4711061160_NC01_START.CATProcess from the directory containing the scenario.

οPart Operation Definition

Right-click the Part Operation node on the PPR tree. Select "Part Operation.1" and then "Definition..." from the context menu, to open Part Operation Definition Panel:

Select the "Machine" Icon to open the "Machine Editor" panel.

 

 

οMachine Editor Panel

Select the "Assign a machine from file selection" Icon

 

 

οAssign a machine from file selection
Select the ZIMMERMANN_FZ30_S840D.CATProduct:

 

 

οNC Resources
Verify / Modify default NC Resource settings:

 

 

οWorkpiece Automatic Mount
Execute the Command "Workpiece Automatic Mount" from the "Machine Management" Toolbar:

Actions done by Delmia on Automount:
a) Attach the Workpiece CATPart/CATProduct to the Machinetool table CATPart/Catproduct
b) Snap/Align the Workpiece CATPart/CATProduct with its NC Axis System to the Workpiece Mount Point of the Machinetool
Explanation:
In this scenario, the Reference Axis from the Part Operation is used for positioning the Set-up-Product with the machine automatically. There is a additional Axis-System (G54) as Origin for the complete Manufacturing Program. The coordinates for the nc-output are calculated from this Axis-System G54.

 

 

 

Run Scenario

This section describes how to run the scenario.

•Load CATProcess

Open document “ZIMMERMANN_4711061160_NC01_START.CATProcess " and follow the steps of Configure Scenario section or open the document  “ZIMMERMANN_4711061160_NC01_END.CATProcess”.

 

 

•Compute Toolpath

Open Manufacturing Program 1 (to be controlled) and compute toolpath for

this Manufacturing progam

Execute the command “Compute Toolpath”.

 

 

•Define Simulation options
Double-Click on the Manufacturing Program or with Mouse-Key 3 (Definition) you can setup the ISO-Code Simulation-Options

ISO Data for simulation is automatically selected:

If there is an entry in the NC File Textfield, delete it with icon .
Define the work piece origin options with the icon

Define the cutter compensation options with the icon

In Panel “Cutter compensation options” use only Switch “Relative user radius”.
Detailed Explanations about Cutter compensation with ISO-Code-Simulation see Cutter Compensation
 
In PPR Tree, the Text on the Program has changed:

 

 

•Start ISO Code Based Simulation
Execute the Command "Starts Machine Simulation"  by pushing the icon from this panel.
Now the Cenit MIK CE is called to generate the simulation data. Delmia changes to the Machine Simulation Workbench and opens the "Process Simulation" Toolbar:

Press "Run" to start the simulation.
 

 

•Simulation and ISO Panel

 

•Material Removal
Depending on options (Tools-Options-Machining-Simulation), rough-material (stock) is displayed and the machine simulation works with material removal computation.
For better performance switch off the material removal computation.

 

 

Clash Definition and Detection

This section describes how to define and how to detect clashes.  The process ZIMMERMANN_4711061160_NC01_START.CATProcess is the process for training, you have to define the clashes.  The ZIMMERMANN_4711061160_NC01_END.CATProcess is for demonstration with predefined values/ clash detection and loaded machine.

There are some bugs in the machine operations “Groove Milling 3” and “Profile Contouring 1”

 

Please Check:

- Approach Macros

- Retract Macro

- Tool-Length (Holder)

- Collision between Machine-Head and Part

- Feedrates (dangerous Rapid Movements)

 

•Define a Clash
Open document “ZIMMERMANN_4711061160_NC01_START.CATProcess " and follow the steps of Configure Scenario section. Define a Clash-Detection with Clearance Distance of 5mm with Machine-Head and Part.

and select (Selection1) the Machine-Head

and select  (Selection2) the complete Setup-Product

and activate this Analyse with and switch the Analysis Level to Highlight (by selecting) and Monitor to on (by selecting)

and activate the visualisation.

 

•Activate advanced analyse-Options
Select  collision list:

- select fault selection (example collision 1)
- select Analyse
- select Apply
- see detailed information in collision-list-panel and in additional 3D-window. You can zoom, rotate, move in this window to see the collision details

 

•Modify operations with clashes
If necessary, modify options, tool holder etc. The Manufacturing Program “Manufacturing Program.2 (ok)” is ok, you can check this options.
