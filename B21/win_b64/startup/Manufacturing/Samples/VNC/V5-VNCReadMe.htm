Introduction

The macro file "V5_VNC_Bridge.CATScript" delivered with the CATIA Manufacturing V5 products enables starting Delmia VNC from the V5 Manufacturing workbench by using the command  "Macro Output To VNC".

This macro file uses 'CATIA Manufacturing VB interfaces' to read and extract data from the V5 Manufacturing Process document in order to generate a 'VNC CLI macro' and launch VNC.

This macro will retrieve the Manufacturing tools from the V5 program and initialize them in VNC.
It creates a VNC workpiece from the stock specified in the Part Operation dialog box.

It initialises the machine and loads the ISO NC code program generated from the V5 program.
The machine name specified in the Machine Editor dialog box must be the VNC machine name (for example, T30_machine).

Note that the syntax of the Manufacturing VB interfaces is subject to change in the next releases.
No compatibility of the present set of Manufacturing VB interfaces is guaranteed with future releases.

Before using the V5-VNC functionality, the following actions needs to be done:

------------
Introduction
------------

The V5-VNC-Bridge.CATScript macro allows to simulate a CATIA manufacturing
process in Delmia's VNC.
After computing your process and generating a NC program, running this macro
will start VNC with your process data (machine, tools, workpiece, program).

Installation
------------

To allow the CATIA simulation in VNC, you have to customize your VNC and CATIA
installation.

Note : we will call DENEB_PATH the root installation path of DENEB products.
       CATIAPATH will be the CATIA root directory (ex: D:\DassaultSystemes\B06).

A) You must create some directories where the demo and simulation data
will be copied.

They are : DENEB_PATH\VNCLib\CLIMACROS\CATIAVNC
  DENEB_PATH\VNCLib\PARTS\CATIAVNC
  DENEB_PATH\VNCLib\PARTS\CATIAVNC\TOOL_HOLDERS
  DENEB_PATH\VNCLib\PROGRAMS\CATIAVNC
 
 

B) Then, you must copy files described in the file list:

-----------------
Environment Setup
-----------------

* Put the path of your DENEB_PATH\vmap\vnc.bat file in the system path, so you can
start VNC while just calling "vnc" from any directory.
 

* Edit the DENEB_PATH\vmap\vnc.bat file and update the DENEB_PATH variable to your
DENEB installation path.
The current value is : set DENEB_PATH=c:\deneb
 

* Edit the DENEB_PATH\vmap\v5\V5setup.bat and update the following variables :

 V5_CATIA_VERSION
 V5_INSTALL_DIR

V5_CATIA_VERSION is the version used to read CATPart or vrml files in VNC.
Ex : set V5_CATIA_VERSION=V5R6
Warning : You must have the CATV5Services.dll file corresponding to the setted version.
-------   Here, for example, DENEB_PATH\vmap\v5\V5R5\CATV5Services.dll

V5_INSTALL_DIR is the installation folder corresponding to V5_CATIA_VERSION
Ex : set V5_INSTALL_DIR=F:\DassaultSystemes\B05

Warning : If your V5_CATIA_VERSION value contains a space character, the VNC program
-------   will not use it correctely.
          So, you will have to create a virtual drive pointing to this folder, and set the
          V5_CATIA_VERSION value to this drive.
          In my exemple : if not exist P:\nul subst P: "F:\Dassault Systemes\B05"
                          set V5_INSTALL_DIR=P:\
 
 

--------------------------------------------
Customizing the V5-VNC-Bridge.CATScript file
--------------------------------------------

Edit the V5-VNC-Bridge.CATScript file and search the string : "Sub CATMain"

At the begining of this procedure, you may have to redefine following variables :

- FolderSeparator : separator character in a path

- theVNCRoot : VNCLib installation folder
               It must be a folder corresponding to the directory config used
               when starting VNC.

- theVNCBatRoot : the path of : DENEB_PATH/vmap/

- theVNCCommand : the command to start vnc in batch mode

- EditMacro : to edit the generated macro when calling VNC
- theTextEditorCommand : command to call a text editor (if you want to edit the macro)
 
 

------------
Runing the demo
------------

A ) Start CATIA.

B)  Check the presence of automation TypeLibs:
Go in popup menu Tools + Options.. + General
Go in Automation tabpage.

Look at the "Selected" column and check wether the following TypeLibs are already present in
the "Selected" column:
MfgTypeLib , PPRTypeLib, ProcessTypeLib, GenKweTypeLib.
If so press "OK" on the Options panel and go to step D .

If one of them is missing,do the following :

select Browse button and go in the CATIAPATH\intel_a\code\bin directory and select the missing one.
Now the selected TypeLib should be present in the "selected" column.

Repeat this action so that the four TypeLibs are present in the "selected" column.
Press "OK" on the Options panel and exit CATIA.

C) Restart CATIA.

D) Open a demonstration process  :
  CATIAVNC-v-mill-3axes-demo.CATProcess or CATIAVNC-T30-machine-demo.CATProcess .

If you want to create the NC program, you can do it by using the CINCINATI_T30_5X_mm
post-processor, otherwise, you may use the supplied program.

E) Call the V5-VNC bridge fom the "Vnc" button or the "Tools"->"Macro"->"Macros..." menu.
You will have to confirm or change the used NC program name.

F) Then, VNC starts and reads the manufacturing process data.
In VNC popup menu use "Run"->"Pendant"->"Cycle Start" menu to see the simulation.
